Input api.kdl
application "Todo" version="0.1" {
    auth provider="User" identity="name" secret="password"
    entity "User" protected=true {
        schema {
            pk "id" type="int"
            field "name" type="string"
            field "email" type="string"
            has-many "Todo" as="tasks" on-delete="cascade"
        }
    }

    entity "Todo" {
        schema {
            pk "ID" type="int"
            field "Title" type="string"
            field "status" type="string"
            field "created_at" type="datetime"
            field "due" type="datetime"
            field "is_done" type="bool"
            field "attachment" type="file" {
                file-options accept=".pdf" \
                max-size=5e3 \
                folder="task_files/"
            }
            belongs-to "User" as="owner"
        }
    }

    profile "dev" default=true {
        listen address="0.0.0.0" port=80
    }
    profile "prod" {
        listen address="localhost" port=80
        listen address="0.0.0.0" port=443 cert="example.crt" key="example.key"
    }
}

Process

Three steps.
That's all.

No scaffolding commands. No generated directories to maintain. No runtime dependencies beyond Terranova itself.

01 / 03
[kdl]
Write a KDL schema
Declare your resources, fields, types, relationships, and auth rules in a single api.kdl file. The syntax is terse and readable — closer to a config file than code.
02 / 03
[run]
Run terranova serve
Point Terranova at your KDL file. It parses the schema, validates it, connects to your database, and starts serving — in under a second.
03 / 03
[api]
You have an API
CRUD endpoints, pagination, filtering, auth middleware, validation, and a live OpenAPI spec are all ready to go. No further configuration required.
$ terranova serve ./api.kdl --port 8080 --db postgres://localhost/mydb
In depth

KDL in.
REST out.

One resource block becomes a full suite of endpoints with proper HTTP semantics, status codes, and response shapes — automatically.

api.kdl
resource "articles" { // field name, type, constraints fields { id "uuid" pk=true title "string" required=true max=120 body "text" author_id "uuid" ref="users" published "bool" default=false created_at "datetime" auto="insert" } // authentication strategy auth "bearer" { read public write "owner" } // query options filter "published" "author_id" sort "created_at" pagination limit=25 max=100 }
terranova
Generated endpoints
GET /articles public
Paginated list. Supports ?published= and ?author_id= filters, ?sort=created_at.
POST /articles bearer
Creates a record. Validates required fields, max-length on title, sets created_at automatically.
GET /articles/{id} public
Fetch by UUID. Returns 404 if not found.
PATCH /articles/{id} owner
Partial update. Auth checks that token subject matches author_id. Returns 403 otherwise.
DEL /articles/{id} owner
Hard delete. Returns 204 No Content on success.

Bearer token, API key, or public — per-resource and per-method. Owner-scoped write guards check claims against record fields without extra code.

Expose query parameters automatically. Declare which fields are filterable and sortable — Terranova validates inputs and constructs safe parameterized queries.

Type coercion, required checks, min/max, regex patterns, and enum membership — all declared in KDL, enforced on every write, returned as structured error bodies.

Reference

KDL syntax cheat sheet.

KDL (KDL Document Language) is a small, human-readable node document format. Terranova adds a thin API-specific vocabulary on top.

Node Example What it does
resource resource "users" { … } Declares a top-level REST resource. Generates a full set of CRUD endpoints under /users.
fields fields { name "string" required=true } Block of field declarations. Each line is name type [attributes…].
pk=true id "uuid" pk=true Marks a field as the primary key. Used in /{resource}/{id} routes.
ref= user_id "uuid" ref="users" Foreign key reference. Enforces referential integrity and enables relational queries.
auto= created_at "datetime" auto="insert" Auto-populates the field on insert or update. Values: "insert", "update", "always".
auth auth "bearer" { write "owner" } Sets the authentication strategy. Optionally configure read/write scopes separately.
filter filter "status" "created_at" Exposes listed fields as query params on the list endpoint.
sort sort "created_at" "name" Fields available for ?sort= ordering on the list endpoint.
pagination pagination limit=20 max=100 Enables cursor/offset pagination. Sets default page size and maximum cap.
